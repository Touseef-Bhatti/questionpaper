Missing values/outliers/duplicates/incorrect labels:

Data Transformation 
– Encoding Categorical Data
Standardizing categorical data: “Male" vs "male“
Categorical Data Encoding
Label Encoding
: Converts each unique category into an integer.
Ordinal Encoding
: Like label encoding but explicitly preserves order.
One-Hot Encoding
: Creates a binary column for each category.
Binary Encoding
: Converts categories into binary numbers and splits them into separate columns.
Target (mean) Encoding
: Replace each category with the mean of the target variable against each category.
Frequency Encoding
: Replace each category with its frequency count or proportion in the dataset.
Count Encoding
: Same as frequency encoding but keeps raw count instead of proportion.
Hashing Encoding
: Uses a hash function to assign categories to a fixed number (2/3/etc.) of columns.

Data Splitting 
Helps preventing overfitting; gives realistic performance estimate; helps hyperparameter tunning; allows fair comparative analysis
Parts
: Training, Test, Validation (optional)
Common split ratio
: 80-20, 70-30, 60-20-20, 60-0-40 in CV
Stratification
: Ensures both sets contain a similar class distribution in case imbalanced dataset.
K-Fold Cross-Validation: 
Instead of one split, the data is split into k equal parts, and the model is trained/tested k times, rotating the test set. It helps reducing variance. Also, used in model selection.
